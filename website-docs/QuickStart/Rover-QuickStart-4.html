4. Running Rover

This section takes a step beyond configuration and shows how to execute rulesets on the remote machines.

4.1 Using the Command Line

The command line can be ran interactively or in batch mode. Running interactively, you will be able to enter password information, select hosts, and modify the current configuration. By default rover will execute in interactive mode, prompting you for your password(s). Other switches that force interactive modes are:

  • -s or --shell: Enter built-in shell
  • -p or --pick: Interactively pick hosts
When entering password information from the command line, you can enter as many passwords as you like. Press enter, leaving the entry blank, in order to finish entering passwords.

In order to avoid being prompted, and run in batch mode, you must not use any of the above switches and also specify the following:
  • --no-auth: Do not prompt for authentication
Here is an example of running rover in batch mode. This example selects the Ruleset Unix_Uptime, specifies there be no password prompting, no banner, and use verbose output: 
$ rover -r Unix_Uptime --no-auth --no-banner -v

4.2 Using the Graphical Interface

When using the graphical interface, you will have to enter your password information manually. To do this, from the main window select File, then Set Password. You will be able to enter passwords for your username, or root, in case you specified logging in as root.

Type in passwords in the text boxes, and click the Add button to included them. A running count of how many passwords you have entered is displayed for both users. Below is a screenshot of the password window:



When your ready to run Rover, select File from the main menu, then Run Rover. The following window will then be displayed:



From this window, you can control whether or not the commands need to be ran as root. You can also override the configured rulesets by using the "Quick Command" feature. See Using Rover Effectively for more details.

When your ready to run, click the "Run" button. A progress bar and cancel button will appear durring the run. Afterwards, each machine that ran will appear in either the Success or Failure window.
